# 去 AI 味设计

## 定位

把本技能当成决策框架，不要当成固定风格预设。参考项目只提供观察证据；不要继承其页面模块、业务题材、色号、字体或构图，除非当前任务也给出了相同理由。

让结果显得由人做出判断，而不是由模型把流行元素平均混合。真实性来自具体内容、真实约束、清楚取舍和可复核细节，不来自故意做旧、随机错位或假装手工。

## 成功标准

- 让设计能回答：给谁看、解决什么、先看什么、为什么采用这种形式。
- 让每个主要元素对应内容、动作、状态、来源或品牌含义。
- 让视觉在删除品牌名后仍保留当前任务的独特证据，而不是适用于任何项目。
- 让图片、文字、界面和交互在真实载体与目标尺寸中成立。
- 让使用的生成式工具不可从套路、错误或虚构事实中被一眼识别。

## 工作流

1. 审计输入：读取内容、品牌、现有实现、真实素材、目标载体、受众与功能约束。
2. 把信息分成三类：必须保留、允许改变、仅供参考。参考图中的模块和样式默认属于“仅供参考”。
3. 写一句设计主张和一句反向主张。例如：“像一本可长期翻阅的研究档案；不要像通用 SaaS 落地页。”
4. 使用分支路由选择一个主分支；混合任务只把其他分支当辅助。
5. 先定义语义化设计变量和四段提示词契约，再编码、排版、生成或编辑。
6. 做最小必要改动，保留现有功能、数据与素材真值。
7. 在目标尺寸、真实内容和降级条件下验证；发现模板化元素时回到主张重新取舍。

把用户明确提出的高饱和、数据密集、无纸张感、品牌字体或指定载体规范放在所有建议范围之前。范围只用于缺少依据时起步，不能覆盖明确需求。

## 分支路由（Branch Router）

| 任务核心 | 主分支 | 判断依据 |
| --- | --- | --- |
| 网站、应用、交互页面、设计系统或响应式组件 | 界面分支 | 输出需要状态、导航、反馈、适配和可访问性 |
| 海报、封面、分享图、信息图、插画或其他固定画布 | 静态分支 | 输出尺寸固定，以构图和阅读顺序为主 |
| 人物、空间、产品、活动或现场，且真实性属于内容 | 实拍照片分支 | 必须保留真实主体、场景和来源关系 |
| 标题、简介、按钮、说明、叙事或语气审查 | 文案分支 | 主要问题是事实、表达和信息密度 |
| 页面同时包含实拍与文案 | 按主要证据路由 | 照片承担证据时走实拍；界面承担任务时走界面 |
| 没有可靠素材却要求“像实拍” | 静态或文案分支 | 使用明确的非纪实表达，不合成虚假证据 |

不要把多个分支的典型效果全部塞进同一输出。摄影、3D、信息图、产品截图和长文案同时争抢焦点，是常见的 AI 拼贴感。

## 边界（Boundaries）

- 不把参考项目的模块、页面顺序、行业语汇、人物资料或品牌资产写成通用标准。
- 不编造人物、场景、客户、奖项、数据、引用、截图、评价或工作成果。
- 不用生成式人物照片冒充本人、团队、活动现场、新闻记录或真实用户。
- 不修改未授权的身份特征、身体、产品结构、Logo、证件、界面数据和关键文字。
- 不复刻参考作品的独特构图或受保护元素；只提炼关系、节奏、材质与判断方法。
- 不为视觉统一破坏现有导航、状态、表单、鉴权、数据模型、语言、排序或业务逻辑。
- 不默认新增框架、字体包、图标包、动画库或运行时依赖；先复用现有能力。
- 不把生成结果直接视为完成；必须检查事实、文字、结构、裁切、无障碍和生成错误。
- 不发布、不推送、不覆盖远程；只有用户明确要求时才执行外部写入。

## 通用设计系统

### 先定义语义变量

按角色定义变量，不从某个参考项目复制值：

- 表面：`canvas / surface / elevated / overlay`
- 文字：`ink / muted / inverse / link`
- 结构：`rule / radius / shadow / grid / gutter`
- 强调：`accent / accent-soft / success / warning / danger`
- 字体：`display / body / label / mono`
- 动效：`duration / easing / distance / stagger`

先解释每个角色的用途，再选择具体颜色、字体和数值。现有品牌变量优先于新建变量。

没有品牌系统时，从任务现场提取 3–5 个视觉线索，例如真实材料、工具、环境、标识体系、阅读距离和受众习惯；根据这些线索选值，不根据本技能的示例或代理偏好选值。

### 留白与主体

- 把留白当成关系，不当成配额。用它分组、停顿、聚焦和保护操作区。
- 先根据内容密度选择紧凑、标准或沉浸三档节奏，再建立一致的间距级别。
- 页面水平边距通常从 `16–24px` 起步，并随视口渐进增加；具体值由设备、语言和内容密度决定。
- 长文单行优先控制在 `45–75ch`；数据表、代码、地图和画布按任务需要扩展。
- 固定画布可让主视觉约占 `30%–60%`、明显留白约占 `25%–60%`，但证件照、目录、仪表盘和数据密集页面不必服从该范围。
- 只保留一个第一焦点。第二焦点必须帮助理解第一焦点，而不是用来填空。
- 不把“主体越小、留白越大”当成高级感公式；在缩略图或移动端失去信息时立即重排。
- 使用光学对齐修正字形、图标和图片边缘，不为追求机械网格牺牲视觉平衡。

### 字体与层级

- 先依据语言、品牌、阅读距离和内容类型选字体，不默认使用某一套无衬线、衬线或等宽字体。
- 正文使用一套高可读字族；展示字或等宽字只有在承担明确角色时才加入。通常不超过 2 个主字族。
- 用字号、字重、行高、字宽和空间共同建立层级，不只靠“大标题 + 小正文”。
- 相邻层级可从约 `1.2–1.6` 的比例开始试排，再根据字形与文案长度调整。
- 正文行高通常落在 `1.45–1.8`；短标签可更紧，长中文与小字号应更松。
- 等宽体只用于代码、编号、时间、状态或元数据；不要把它当成自动的“设计感”。
- 逐字检查中文、英文、数字、标点、大小写与换行；必须准确的文字优先使用代码或排版工具生成。

### 颜色

- 先确定明暗关系与对比，再确定色相。颜色必须承担品牌、层级、状态或强调之一。
- 从一组中性色和一个主强调色开始；只有状态或分类确有需要时才增加颜色。
- 不把暖灰、纯黑、蓝紫渐变、印章红或任何现成色板当成通用默认。
- 大面积强调色使用低强度变体；高饱和色保留给小面积焦点、状态和反馈。
- 渐变必须说明用途，例如表现光源、深度或状态过渡；无法说明时使用纯色。
- 在正常、悬停、禁用、错误、暗色和图片背景上分别检查对比，不靠低对比灰字显得高级。

### 材质、边界与光影

- 每个输出最多选择一个主材质隐喻，例如纸、屏幕、印刷、胶片、金属或自然光。
- 纹理只用于支持来源和触感；不要靠噪点、折痕、胶带、指纹或错版伪装成人工制作。
- 用边框、底色、阴影、重叠或留白中的少数几种表达层级，不要全部叠加。
- 阴影对应真实的层级或遮挡；没有悬浮关系时优先用边框或底色分层。
- 圆角由产品语气、尺寸和交互决定。不要让所有容器、图片、按钮和标签共享同一个夸张圆角。

### 图像与图形

- 优先使用与内容直接相关的实拍、产品画面、文档、界面、数据或品牌资产。
- 每张图只设一个视觉主张；附加图标和装饰必须带来新的信息。
- 让系列图片共享系统，而不是共享同一套中心构图、机器人、发光线和漂浮符号。
- 对含准确文字、Logo、图表或界面的画面，优先使用 HTML / CSS / SVG 或专业排版工具；生成模型只负责适合生成的局部素材。
- 为所有有意义图片提供替代文本；装饰图应隐藏于辅助技术。

### 动效与响应式

- 让动效说明进入、退出、状态变化、层级或空间关系；不为“有生命力”添加永动漂浮。
- 微交互通常从 `120–240ms` 起步，页面级过渡从 `240–600ms` 起步；编辑式开场可更慢，但不得阻塞核心内容或缺少跳过方式。
- 控制位移、模糊和错峰数量；一个视区只保留一个持续性环境动效。
- 支持 `prefers-reduced-motion`，关闭动画后内容、顺序和操作必须完整。
- 在窄屏重新编排优先级，不把桌面布局等比缩小。先改单列、缩短次要文案、调整裁切，再考虑隐藏内容。
- 验证项目实际断点与代表性设备，不把某个固定宽度写成所有项目的标准。

## 界面分支

1. 先列出主要任务、内容清单、状态、入口和退出路径，再画页面结构。
2. 让信息架构决定容器和网格；不要为了套用 Bento、双栏 Hero 或侧边档案栏而拆碎内容。
3. 先完成默认、加载、空、错误、禁用和成功状态，再加装饰和动画。
4. 复用现有组件与主题变量；新增视觉规则应减少分叉，而不是制造新的局部系统。
5. 用渐进披露处理复杂信息：首层给判断与动作，详情层给证据与长内容。
6. 检查键盘导航、焦点、触控尺寸、语义标签、对比、文本缩放、溢出和 reduced motion。

## 静态分支（Static Branch）

1. 明确载体、尺寸、比例、裁切安全区、观看距离和文字真值。
2. 只有栅格参考时可用 OCR 辅助识字，但用户原文与可编辑源文件才是文字真值。
3. 先用黑白块面确定第一焦点、阅读顺序和留白，再选择颜色和材质。
4. 把必须准确的文字、Logo、界面和图表单独排版，不让生成模型猜测。
5. 限制符号数量；流程、数据和插图只保留完成叙事所需的节点。
6. 在缩略图、实际尺寸、目标裁切和灰度模式下分别检查一次。

## 实拍照片分支（Live Photo Branch）

1. 只使用用户提供、现场拍摄或来源明确且可使用的照片；记录主体、场景和用途。
2. 没有真实素材时，改用明确标注的示意图、授权图库、静态排版或占位，不合成纪实证据。
3. 根据动作、视线、环境关系和叠字区裁切，不默认居中、虚化背景或把人物放到固定比例。
4. 保留身份、面部结构、手部、衣物、产品结构、Logo、物件数量、光源和空间关系。
5. 一次只改一个变量；局部修复后对照原图检查未授权变化。
6. 检查皮肤与材质纹理、手指、牙齿、眼镜、反射、直线、重复物件、光源和裁切。

## 文案分支

- 从事实、动作、对象、限制和结果出发；让内容能被来源逐项证明。
- 用具体名词和动词替代愿景、情绪与抽象价值词。
- 一句话承担一个主要判断；句长和节奏可以变化，不硬凑三段排比。
- 文案长度由载体和层级决定，不把某个 Hero、按钮或卡片字数写成普遍规则。
- 保留领域中真正必要的术语，同时解释陌生概念；不要用英语标签制造专业感。
- 没有可信数字时不要写百分比、倍数、领先、显著提升或专家级。
- 让按钮直接描述动作，例如“查看方法”“下载报告”“继续编辑”。

## 四段提示词契约（Four-Paragraph Prompt Contract）

每次设计、生成或编辑前先写四段完整约束。四段可以简短，但不得省略事实与反向约束。

1. **用途与事实**：说明载体、受众、任务、素材来源、现有系统和必须准确的内容。
2. **结构与构图**：说明第一焦点、阅读或操作路径、信息密度、主体关系、留白和响应式变化。
3. **视觉与交互系统**：说明字体角色、颜色角色、材质、图像来源、边界、反馈和动效理由。
4. **边界与验收**：说明必须保持不变的事实、禁止加入的套路、输出格式、目标尺寸和验证方式。

### 静态分支四段示例

```text
为一份城市水质研究制作 A4 竖版封面，读者是社区居民。只使用已提供的报告标题、日期、采样地点和真实数据表；所有文字和数字必须准确。

标题与日期形成第一层，采样地图局部作为第二层，数据表只露出一处可读证据。保留足够装订边与呼吸区，不使用居中的奖章式构图。

从报告现有蓝色与纸张照片取样建立颜色角色，正文沿用报告字体；使用一条细线连接地点与数据。材质保持清晰印刷感，不添加无来源的实验室器材或科技光效。

不得生成伪地图、伪数据、漂浮仪表盘、发光粒子、乱码、假 Logo 或不存在的机构。输出印刷 PDF 与预览 PNG，并核对文字、出血、灰度层级和缩略图。
```

### 实拍照片分支四段示例

```text
使用用户提供的面包店清晨实拍制作首页横幅。照片中的店员、烤箱、面包和空间关系都是真实内容；只加入店名、营业时间和一个预订入口。

沿店员动作方向裁切，让操作台成为环境证据，并在原本较暗的空区放文字。保持现场感，不把人物移到正中，也不制造额外顾客或道具。

保留自然窗光、面粉痕迹和金属烤盘质感，只做小幅白平衡与曝光调整。字体与门店招牌协调，界面控件保持清楚，不覆盖人物双手和食品。

不得磨皮、换脸、重做手指、增加蒸汽、假阳光、假景深、霓虹轮廓或不存在的商品。输出响应式裁切，并检查身份、手部、反射、招牌文字和移动端焦点。
```

## 反向约束与强制规避项（Hard Avoids）

把以下内容当成反向检查，不当成绝对素材禁令。题材本身确实包含机器人、齿轮、渐变、纸张或 3D 时可以使用，但要说明它与内容的直接关系，并避免与其他套路成套堆叠。

### 不把“去 AI 味”做成另一套模板

- 不默认使用暖灰纸张、印章红、深色编辑封面、等宽微标签或某组固定字体。
- 不强制非对称、超大留白、小主体、粗粝纹理、手写批注、胶带和错版效果。
- 不把所有网站做成巨大 Hero、Bento 卡片、浮动胶囊导航或宣言式页脚。
- 不从参考项目搬运个人资料、履历、地图、质量流程、生活模块或行业题材。
- 不为了“真实”故意降低完成度；错误、脏污和随机不一致不是人的判断。

### 避免典型生成式视觉堆栈

- 无理由的蓝紫渐变、发光边缘、赛博网格、粒子宇宙和数据流。
- 塑料 3D 人物、黏土机器人、玻璃球、软糯气泡和悬浮设备。
- 中心勾选、同心圆轨道、四色节点与四周漂浮卡片的标准信息图。
- 放大镜、盾牌、齿轮、火箭、灯泡、拼图、奖杯和大脑图标的无差别组合。
- 全部元素等距对称、每块都有阴影、每张卡都有渐变、每个角落都被填满。
- 伪代码、伪仪表盘、乱码、假 Logo、不可解释的数据和重复物件。
- HDR、橙青滤镜、假景深、蜡质皮肤、错误手指和统一微笑的团队照。

### 避免典型生成式文案

- `在这个快速变化的时代`、`开启全新篇章`、`探索无限可能`、`让每一次……都……`。
- 无依据的 `赋能`、`重塑`、`颠覆`、`焕新`、`无缝`、`极致`、`全方位`、`一站式`。
- 连续使用 `不止是 A，更是 B`、`从 A 到 B`、`让 A 变得 B` 等模板句。
- 三段等长排比、宏大开头、口号结尾和没有信息作用的英文副标题。
- 用 `AI 驱动`、`智能化`、`创新` 代替工具、输入、动作和产物。
- 没有证据的效率百分比、行业领先、专家级或用户喜爱。

## 验证清单

### 通用性

- 技能输出没有继承参考项目的专属模块、人物、行业、色号、字体或文案。
- 每个固定值都能由载体、品牌、可访问性或内容密度解释；否则改成变量或删除。
- 更换项目后，规则仍能工作，但结果不会长得一样。

### 视觉与界面

- 缩小、灰度和去掉装饰后，第一焦点与阅读顺序仍成立。
- 每个颜色、图标、卡片、纹理和动效都有信息或交互职责。
